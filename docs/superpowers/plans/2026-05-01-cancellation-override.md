# Admin Cancellation Override Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let an admin mark a HashTracks `Event` cancelled with a required reason; the override survives every subsequent merge and reconcile pass; un-cancel is one click; audit history is durable across re-cancel cycles. Public-facing UX unchanged.

**Architecture:** Three new nullable current-state fields plus one append-only JSON audit-log field on `Event`. A centralized `isAdminLocked()` helper in `src/pipeline/merge.ts` guards **both** restore sites that flip `CANCELLED → CONFIRMED`. Server actions in `src/app/admin/events/actions.ts` reuse `getAdminUser()`, `appendAuditLog()` from `src/lib/misman/audit.ts`, and the existing revalidation/toast pattern. Admin UI extends the existing `EventTable` row-action dropdown with a confirmation dialog. Public-facing rendering and filters are untouched.

**Tech Stack:** Prisma 7 + PostgreSQL (Railway prod, local Postgres 17 for `migrate dev`); Next.js 16 App Router; TypeScript strict; Vitest; shadcn/ui (`Dialog`, `DropdownMenu`, `Textarea`, `Tooltip`); `sonner` toasts; Clerk auth via `getAdminUser()`.

**Spec:** [`docs/superpowers/specs/2026-05-01-cancellation-override-design.md`](../specs/2026-05-01-cancellation-override-design.md). Read it once before starting; this plan is the execution-level companion.

```ts
/**
 * Admin override: mark an event CANCELLED with a required reason. The override
 * survives all subsequent merge / reconcile passes via the merge pipeline's
 * `isAdminLocked` guard. Un-cancel via `uncancelEvent`.
 *
 * See docs/superpowers/specs/2026-05-01-cancellation-override-design.md.
 */
export async function adminCancelEvent(
  eventId: string,
  reason: string,
): Promise<ActionResult<{ kennelName: string; date: string }>> {
  const admin = await getAdminUser();
  if (!admin) return { error: "Not authorized" };

  const trimmed = reason.trim();
  if (trimmed.length < 3) return { error: "Reason must be at least 3 characters" };
  if (trimmed.length > 500) return { error: "Reason must be 500 characters or fewer" };

  const event = await prisma.event.findUnique({
    where: { id: eventId },
    include: { kennel: { select: { shortName: true, slug: true } } },
  });
  if (!event) return { error: "Event not found" };
  if (event.adminCancelledAt) {
    return { error: "Event already admin-cancelled — un-cancel first to change reason" };
  }

  const auditEntry: AuditLogEntry = {
    action: "cancel",
    timestamp: new Date().toISOString(),
    userId: admin.clerkUserId,
    changes: { status: { old: event.status, new: "CANCELLED" } },
    details: { reason: trimmed },
  };

  await prisma.event.update({
    where: { id: eventId },
    data: {
      status: "CANCELLED",
      adminCancelledAt: new Date(),
      adminCancelledBy: admin.clerkUserId,
      adminCancellationReason: trimmed,
      adminAuditLog: appendAuditLog(event.adminAuditLog, auditEntry),
    },
  });

  console.log("[admin-audit] adminCancelEvent", JSON.stringify({
    adminId: admin.id,
    action: "admin_cancel_event",
    eventId,
    kennelName: event.kennel.shortName,
    eventDate: event.date.toISOString(),
    reason: trimmed,
    timestamp: new Date().toISOString(),
  }));

  revalidatePath("/admin/events");
  revalidatePath("/hareline");
  revalidateTag(HARELINE_EVENTS_TAG, { expire: 0 });
  revalidatePath(`/hareline/${eventId}`);
  revalidatePath(`/kennels/${event.kennel.slug}`);

  return {
    success: true,
    kennelName: event.kennel.shortName,
    date: event.date.toISOString(),
  };
}
```

Note the `revalidatePath(/kennels/${event.kennel.slug})` — Codex's review caught that the original design used `kennelId`, which is the wrong route shape. The `slug` is loaded via the `include` clause.

```ts
/**
 * Restore a CANCELLED event back to CONFIRMED. Works for both reconciler-set
 * and admin-set CANCELLED. Clears any admin-override lock fields and, if the
 * event was admin-cancelled, appends an "uncancel" entry to the audit log.
 */
export async function uncancelEvent(
  eventId: string,
): Promise<ActionResult<{ kennelName: string; date: string }>> {
  const admin = await getAdminUser();
  if (!admin) return { error: "Not authorized" };

  const event = await prisma.event.findUnique({
    where: { id: eventId },
    include: { kennel: { select: { shortName: true, slug: true } } },
  });
  if (!event) return { error: "Event not found" };
  if (event.status !== "CANCELLED") return { error: "Event is not cancelled" };

  const wasAdminCancelled = event.adminCancelledAt !== null;
  const auditEntry: AuditLogEntry | null = wasAdminCancelled
    ? {
        action: "uncancel",
        timestamp: new Date().toISOString(),
        userId: admin.clerkUserId,
        changes: { status: { old: "CANCELLED", new: "CONFIRMED" } },
      }
    : null;

  await prisma.event.update({
    where: { id: eventId },
    data: {
      status: "CONFIRMED",
      adminCancelledAt: null,
      adminCancelledBy: null,
      adminCancellationReason: null,
      ...(auditEntry !== null
        ? { adminAuditLog: appendAuditLog(event.adminAuditLog, auditEntry) }
        : {}),
    },
  });

  console.log("[admin-audit] uncancelEvent", JSON.stringify({
    adminId: admin.id,
    action: "uncancel_event",
    eventId,
    kennelName: event.kennel.shortName,
    eventDate: event.date.toISOString(),
    wasAdminCancelled,
    timestamp: new Date().toISOString(),
  }));

  revalidatePath("/admin/events");
  revalidatePath("/hareline");
  revalidateTag(HARELINE_EVENTS_TAG, { expire: 0 });
  revalidatePath(`/hareline/${eventId}`);
  revalidatePath(`/kennels/${event.kennel.slug}`);

  return {
    success: true,
    kennelName: event.kennel.shortName,
    date: event.date.toISOString(),
  };
}
```

Two real changes vs the original:

1. Loads `kennel.slug` and adds `revalidatePath(/kennels/${event.kennel.slug})`. Bonus fix per Codex's review.
2. Clears all three lock fields and conditionally appends an audit entry only when the event was admin-cancelled.
