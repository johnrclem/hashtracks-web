## Adapter notes / new-scraper plan

**New `MadridHashAdapter` (`src/adapters/html-scraper/madrid-hash.ts`), modeled on `ewh3.ts`.** Both
are self-hosted-WordPress trail-post scrapers. Madrid is actually *simpler* than EWH3: all run data
is **labeled in the post body**, one clean run per post (no embedded "Hash Trash" recaps), and a
decimal GPS pair is provided.

Shape (ILLUSTRATIVE — verify field/import names against the live repo before writing):

```text
fetch(source, options?):
  baseUrl = source.url || "https://madridhhh.com/"
  days = options?.days ?? source.scrapeDays ?? 365
  wpResult = await fetchWordPressPosts(baseUrl, 30)   // perPage ~20–30 covers all future + recent past
  if wpResult.error → return { events: [], errors: [error.message] }   // fail loud
  events = []
  for post of wpResult.posts:
     $ = cheerio.load(post.content); $("p, br, h1, h2, h3, h4").before("\n"); body = $.text()
     ev = parseMadridRunBody(body, post.title, post.url)   // null if body has no "Run No.:" → skip non-run posts
     if ev: events.push(ev)
  // FAIL-LOUD guard: if wpResult.posts.length > 0 && events.length === 0 → push an error
  //   (brand-new source has no fill-rate baseline; a body-format drift must surface, not silently 0)
  return applyDateWindow({ events, errors }, days)

parseMadridRunBody(body, postTitle, url):
  // require a run marker — this is the run/non-run filter
  runNo = body.match(/Run\s*No\.?:\s*([\d.]+)/i);  if !runNo → return null
  date = chronoParseDate( stripWeekday( body.match(/\bDate:\s*(.+)/i)[1] ) )  // authoritative, NOT post.date
  if !date → return null
  startTime = firstHHMM( body.match(/\bTime:\s*(.+)/i)?.[1] )                 // "13:00" / "20:00"
  location = clean( body.match(/\bLocation:\s*(.+)/i)?.[1] )
  hares = body.match(/\bHares?:\s*(.+)/i)?.[1] → split on & / , , sort, join ", "
  coords = parseMadridGps( body.match(/\bGPS:\s*(.+)/i)?.[1] )               // decimal pair; see below
  locationUrl = body.match(/maps\.app\.goo\.gl\/\S+/)?.[0]
  theme = parseMadridTheme(postTitle)   // optional; else leave title undefined → merge synthesizes
  return {
    date, kennelTags: ["madrid-h3"], runNumber: Math.floor(Number.parseFloat(runNo[1])),
    title: undefined /* or theme */, hares, location, startTime,
    latitude: coords?.lat, longitude: coords?.lng, locationUrl,
    sourceUrl: url, description: theme,
  }

parseMadridGps(s):   // s = `[40°27'15.7"N 3°37'45.7"W] or [40.454352, -3.629372]`
  m = s.match(/\[\s*(-?\d{1,2}\.\d+)\s*,\s*(-?\d{1,3}\.\d+)\s*\]/)   // the DECIMAL bracket (order: lat,lng)
  return m ? { lat: Number.parseFloat(m[1]), lng: Number.parseFloat(m[2]) } : undefined
```

## Implementation gotchas (for Claude Code — repo knowledge)
Carry these into the build; each has caused a follow-up fix before:
- **`config.upcomingOnly: true` is set** because the forward adapter fetches only the latest ~30
  posts (not the whole archive each scrape) while the backfill owns the deep history — `upcomingOnly`
  restricts `reconcile.ts` to future dates so it won't false-`CANCEL` the aged backfilled archive.
- **Date: use the in-body `Date:` line, NOT `post.date`.** Posts publish ~4 days BEFORE the run
  (e.g. the #2713 post is dated 2 Jun for a 7 Jun run) — `post.date` would mis-date every event.
- **Coords are clean per-run → no `dropCachedCoords` needed.** (Only add it if you ever see a repeated
  default pin, which these do not.)
- **Single-block / fail-loud guard:** a brand-new source has a 0 fill-rate baseline, so emit an
  explicit error when `posts.length > 0 && events.length === 0` (body-format drift must surface, not
  scrape "successfully" with `events: []` and let reconcile run on nothing).
- **Sonar S5852/S5843:** the body label regexes are simple (`/\bHares?:\s*(.+)/i` etc.) — keep them
  that way (single `\s`, no stacked optional groups). For the date, match loose and let
  `chronoParseDate` validate (don't enumerate month-name alternations). Use `Number.parseInt/parseFloat`,
  `String.replaceAll`, `RegExp.exec` for capture-heavy matches, no negated ternaries.
- **http/https:** the site is https — no S5332 http-literal hotspots expected.
- **Self-host the logo** into `public/kennel-logos/madrid-h3.<ext>` (literal `<ext>` placeholder until
  magic bytes confirm — the candidate is a `.bmp` OR `.png`; verify, don't assume).
