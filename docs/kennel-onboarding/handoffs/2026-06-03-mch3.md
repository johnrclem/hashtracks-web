**5. `COUNTRY_INFERENCE_RULES`** — 🔴🔴 **MUST guard against the "New Mexico" collision.** The
rules run on **region names** and default to `"USA"`. A naive `\bmexico\b` rule would also match

lines together at the end of the array, guard first:
```ts

[/\bm[eé]xico\b|\bcdmx\b|ciudad de m[eé]xico/, "Mexico"],     // "Mexico City"/"México"/"CDMX" → Mexico
```

  ```bash
  # expect: Mexico  USA  USA  Mexico
  ```
- (Alternative if you prefer no separate guard line: a single
  `/(?<!new\s)\bm[eé]xico\b|\bcdmx\b|ciudad de m[eé]xico/` lookbehind also works on Node 20 — but
  the two-line guard above is clearer and Sonar-safe. Either is fine; verify with the command above.)

## Implementation gotchas (for Claude Code — repo knowledge, not source knowledge)
- 🔴 **`COUNTRY_INFERENCE_RULES` New-Mexico collision** — the #1 trap here. A bare `\bmexico\b`
  rule breaks Albuquerque (US "New Mexico" → would infer Mexico). Use the New-Mexico→USA guard
  line BEFORE the Mexico rule (or the lookbehind alternative); verify with the `inferCountry` snippet
  above. This is the same class of bug that bit ONH3 (Kenya→USA) and the WSH3 metro regex.
- 🔴 **Meetup group `foundedDate` (2022-09-08) ≠ kennel founding (1983)** — already handled in the
  seed (`foundedYear: 1983`). Don't let a future enrichment pass overwrite it with the Meetup value.
- **`config.upcomingOnly: true` is REQUIRED** — the Meetup events page is a rolling upcoming-only
  window, so without it `reconcile.ts` would CANCEL the backfilled past runs (#746–#756) the moment
  they age off the live feed. (Same rule that ONH3/Bali needed.)
- **`friendlyKennelName` check** — `shortName: "Mexico City H3"` is **>4 chars**, so the function
  short-circuits to it directly (safe — the H7 verbose-title trap only fires for ≤4-char shortNames).
  Still worth a 5-second confirm:
  ```bash
  npx tsx -e 'import {friendlyKennelName} from "./src/pipeline/merge"; console.log(friendlyKennelName("Mexico City H3","Mexico City Hash House Harriers"));'
  # expect: "Mexico City H3"
  ```
- **Self-host the tokenized Meetup logo** into `public/kennel-logos/mch3.<ext>` — the
  `secure.meetupstatic.com/.../highres_506902414.jpeg` URL rotates. **NEVER pre-fill the extension**
  — confirm via `curl -sI` Content-Type AND magic bytes (Meetup usually JPEG → `\xff\xd8`, but verify).
- **Slug is clean ASCII** — `toSlug("Mexico City H3")` = `mexico-city-h3` (no accent, no override
  needed). The kennel page will live at `/kennels/mexico-city-h3`.
- **Backfill = frozen JSON + dumb loader, no committed parser** (Asunción/H7 durable pattern). The
  10-run dataset is already curated above. Dates → UTC noon; idempotent upsert keyed by kennel+date;
  require numeric run# and `date < today`.
- **Sonar:** the new inference regex is simple (single `\s`, no nested quantifiers); the
  New-Mexico-guard two-line form avoids lookbehind entirely. `Number.parseInt(s,10)` /
  `s.replaceAll(...)` / no negated ternaries if you write any loader helpers.
