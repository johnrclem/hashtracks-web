## Source-discovery note (technique used this run — worth reusing)

The four top `queued` targets were all dead this run, so Shanghai H3 was found by turning the existing
**Harrier Central adapter into a live-kennel finder**: replicate the `generateAccessToken("getEvents")`
token (`src/adapters/harrier-central/token.ts`) in a browser page-context `fetch` and POST
`{publicHasherId, accessToken, queryType:"getEvents", cityNames:"<City>"}` to the HC PortalApi. It
returns `[[events]]` with `publicKennelId`, `kennelName`, `kennelUniqueShortName`, city, dates, coords
and logo — i.e. a ready-to-paste config-only source for any HC kennel in an uncovered city. A sweep of
~120 uncovered cities surfaced Shanghai H3 (`63e3cccd-…`) and confirmed Taiwan H3 (`f1330d14-…`, already
seeded) as the only mainland/Taiwan HC hits; most other HC kennels are in already-covered cities. The
unfiltered call errors ("No query constraints") — you must pass `cityNames` (or a kennel/GUID filter).
