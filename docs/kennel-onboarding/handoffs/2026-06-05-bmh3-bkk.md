## Implementation gotchas (for Claude Code — repo knowledge, not source knowledge)
- **`config.upcomingOnly: true` is required** — the forward hareline ages out (completed runs move to the
  year archive), so without it `reconcile.ts` would false-`CANCEL` runs the moment they drop off the forward
  pages. (Same as Bali Hash 2 / ONH3.)
- **Year-less dates + Dec→Jan rollover** — `DD MMM` has no year. Anchor on a reference date and roll a Jan/Feb
  row forward to next year on the forward page. In the backfill, the year is the page's year (`Index20NN.html`).
- **Strip the `AGM` token** before date parse (`"2 Nov AGM"`) and tolerate it appearing in the Hare cell in the
  archive (`**AGM** Codpiece`). Don't let it leak into `hares`/`date`.
- **`TBA` is not data** — map `"TBA"` hare/location to `undefined`; never persist the literal string.
- **Run-number gaps are legitimate** (e.g. #2238/#2239 unassigned) — do not synthesize missing runs; just emit
  the rows present.
- **Tests:** `vi.spyOn(globalThis, "fetch")` accumulates `.mock.calls` across `it()` blocks — add
  `beforeEach(vi.restoreAllMocks)` for accurate per-test counts.
- **Sonar S3776 ≤ 15 / S5852** — extract helpers (`parseHarelineRow`, `parseNextRunBlock`, `inferYear`); for
  dates prefer a loose `\b\d{1,2}\s+\w+\b` + `chronoParseDate` over enumerated month alternations.
- **Code-style:** `Number.parseInt(s, 10)`, `s.replaceAll(...)`, `RegExp.exec()` for capture-heavy use,
  no negated ternaries.
- **`friendlyKennelName("Bangkok Monday H3", fullName)`** is >4 chars → returns shortName verbatim → titles
  "Bangkok Monday H3 Trail #N" (no need to run the short-name check, but the result is confirmed clean).
