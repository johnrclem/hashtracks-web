## Implementation gotchas (for Claude Code — repo knowledge, not source knowledge)
- **🔴 Big5 decode before parse** — fetch bytes, `iconv-lite`/`TextDecoder("big5")`, unit-test that
  "新北捷兔" round-trips. This is HashTracks' first Big5 source. Append the learning to `source-platform-notes.md`.
- **Year from the URL filename, not `new Date()` rollover** — `run_site_2026.htm` → 2026. The adapter
  must also construct the *current* year's URL at fetch time so it doesn't pin to 2026 forever (see the
  year-in-URL SOURCE-url gotcha above).
- **`config.upcomingOnly: true` is set** — the year page carries both past + future 2026 runs (and the
  backfill adds 2013–2025); without it `reconcile.ts` would false-`CANCEL` the past rows as they age.
- **`friendlyKennelName` check** — `shortName: "New Taipei H3"` is >4 chars → short-circuits cleanly to
  `"New Taipei H3"` → titles render `"New Taipei H3 Trail #N"`. (Confirm if you change the shortName.)
- **Strip PII phones from `hares`** — archive rows carry `0xxx-xxx-xxx` / `(02)…` numbers; the frozen
  backfill MUST scrub them too (validation checklist above).
- **Dedupe the highlighted "Important Events" rows vs the weekly list by run number** before emitting
  (#700/#708/#714 appear twice on the page).
- **Multi-day specials** — a `MM/DD~DD` date range (e.g. archive 2013 #46 `11/15~17`, no hare) is one
  event (use the first day), not a parse error; the run-continuity validator must whitelist it.
- **http-only source → Sonar S5332 hotspots.** The kennel `website`, source `url`, and adapter
  `DEFAULT_URL` are `http://` (the site serves no https). Use `https://` literals in mocked tests
  (tests mock `safeFetch`); mark the ~3 real production `http://` literals **SAFE** via the SonarCloud
  REST API (`POST api/hotspots/change_status` with `SONARQUBE_TOKEN` from the *main repo* `.env`) — the
  `update_hotspot_status` MCP tool is broken (per the AH3-NZ retro).
- **Sonar S5852 / S5843** — keep date/phone/run regexes simple: `DATE_RE = /(\d{1,2})\/(\d{1,2})/`,
  strip phones with narrow patterns, no stacked `\s*`-around-alternation. Prefer `Number.parseInt(s, 10)`,
  `s.replaceAll(...)`, `Map.get()` over `Record[var]` (Codacy `detect-object-injection`), no negated ternaries.
