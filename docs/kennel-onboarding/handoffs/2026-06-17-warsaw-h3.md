## Adapter notes / new-scraper plan

**NEW `WarsawH3Adapter` — small static Cheerio over the Mobirise home page.** Not config-only: the page
mixes two shapes (a detailed "next run" block + a plain "Upcoming runs" list), and no existing
config-driven adapter can split a multi-line run block into fields. Model the control flow on
`src/adapters/html-scraper/manila-h3.ts` (single SSR run block, fail-loud guard, `upcomingOnly`) and
`src/adapters/html-scraper/bangkok-monday-hash.ts` (merge a "next run" detail block onto a list by run
number — but **simpler here: one page, no year inference**).

Parsing plan:
- **Fetch:** `fetchHTMLPage("https://warsawh3.com/")` (static; `safeFetch` under the hood — NOT raw `fetch`).
- **Linearize:** Mobirise wraps each content block in `<section class="mbr-section …">` with rotating
  inner classes → key the parser on **visible text**, not selectors. `stripHtmlTags(html, "\n")` →
  split into lines (same helper Manila/NSWHHH use).
- **Next-run block:** find the line matching `/WH3 Run #(\d+)/i` (run #1643). The following lines carry
  `Sat 20 June 2026, 14h00` (date + time), a `Where?` label then the venue line, and a `Who?` label then
  the hare line. Extract: runNumber, date (carries year — parse straight to **UTC noon**), `startTime`
  (`14h00` → `"14:00"`), venue, hare.
- **Upcoming list:** under the "Upcoming runs" heading, each run is two lines: `#1644 July 4, 2026` then
  `Hare: Chasing Yanks`. Loop: `/#(\d+)\s+([A-Z][a-z]+ \d{1,2}, \d{4})/` for run# + date; next `Hare:` line
  for the hare. These rows have no venue/time → leave `location`/`startTime` undefined.
- **Merge by run number**, then dedupe (the next-run #1643 only appears in the detail block; #1644–46 only
  in the list — no overlap here, but dedupe defensively).
- **Hares hygiene:** strip placeholders → `undefined`: `???`, `It Could Be You!`, `TBA`, `Hare needed`.
- **`title`:** leave `undefined` (no themes) → merge synthesizes `"Warsaw H3 Trail #N"`.
- **🔴 Fail-loud zero-row guard** (single-page adapter): `if (rows.length === 0) errors.push("Warsaw H3:
  no run rows parsed — Mobirise markup may have changed")`. Without it a markup drift makes the scrape
  "succeed" with `events: []` and the zero-event health alert never fires on a brand-new source.
- **Registry:** add a `htmlScrapersByUrl` entry keyed on `warsawh3.com` → `WarsawH3Adapter`.
- **Regex safety:** dates carry the year and are simple (`Month D, YYYY` / `D Month YYYY`) — use two small
  regexes (a `#\d+` run regex + a loose `\b\d{1,2}\s+\w+\s+\d{4}\b` or `\w+ \d{1,2}, \d{4}` date), validate
  via chrono-node / `Date.UTC` round-trip. Keep `\s` single (no stacked `\s+` around alternations → S5852).
