### Adapter: `src/adapters/html-scraper/mijas-hash.ts` (NEW)
Model on `src/adapters/html-scraper/barnes-hash.ts` (simple single-kennel UK; uses
`chronoParseDate(text, "en-GB", …)`) and `london-hash.ts` (run-list parsing). Static HTML →

**Parsing plan — each hareline line is one event of the form:**
```
<runNum> - <DD Month YYYY> - <hares?> - <theme?>
e.g.  2020 - 31 May 2026 - Shaggy & AguaSex - AGM Run
      2025 - 28 June 2026                            (no hares, no theme)
      2023 - 20 June 2026 - - Glampout Weekend       (empty hares slot)
```
Tokenizer (split on `" - "`, then position-map; rejoin index ≥3 so a theme containing `" - "` survives):
- **runNum:** leading integer; strip a trailing `a`/`b` (away-weekend sub-runs like `1999a`/`1999b`
  share base #1999 — store the integer; kennel+date dedup keeps the two days distinct).
- **date:** `DD Month YYYY` → parse with `chronoParseDate(seg, "en-GB", ref, { forwardDate:true })`,
  then normalize to **UTC noon**. Trust the per-line date (NOT the `<h2>` month heading — DOM order
  is unsorted; see Live-verification note).
- **hares:** split on `&` / `,`; trim; sort before `join(", ")` for fingerprint stability (idempotency).
- **theme → `title`:** use only when it's a real theme ("AGM Run", "Bull Moon Run", "CCI Elvis
  Birthday Run"); when the slot is empty/whitespace leave `title` **undefined** so `merge.ts`
  synthesizes `"Mijas H3 Trail #N"`. Never let a hare name become the title.
- **location / startTime / coords / endTime:** not present in the hareline → leave undefined.
- Emit `kennelTags: ["mijash3"]` on every event. Honor `options.days` via `buildDateWindow`.

## Implementation gotchas (for Claude Code — repo knowledge, not source knowledge)
- **DOM order is NOT chronological** — parse each line's own `DD Month YYYY`; never use the month
  `<h2>` heading for the year or assume sorted source order (August precedes May on the live page).
- **`config.upcomingOnly: true` is set** — the hareline is a rolling current-year schedule that will
  prune old months over time; this keeps `reconcile.ts` from false-cancelling future rows once past
  months drop off. (Full-archive sources don't need it; a rolling hareline does.)
- **Sort hares before `join(", ")`** — nondeterministic split order otherwise yields fresh
  fingerprints each scrape (Seletar dup-RawEvent bug, PR #541).
- **Sonar S5852 (regex backtracking)** — keep the line tokenizer simple. Prefer
  `line.split(/\s+-\s+/)` over one mega-regex; for the run number use `/^(\d+)[ab]?\b/`, for the
  date a plain `D Month YYYY` search. Avoid `(.+)$`; if needed use `(\S.*)$`.
- **Sonar S3776 (cognitive complexity ≤ 15)** — extract helpers (`parseHarelineLine`,
  `parseRunNumber`, `parseHares`) rather than inlining; the per-line tokenizer + guards exceed 15 inlined.
- **Self-host the tokenized Squarespace logo** into `public/kennel-logos/mijash3.gif` — don't
  reference the rotating `images.squarespace-cdn.com/.../<hash>/logo.gif` URL.
- **New-country region edits are all-5** — region seed alone is insufficient;
  `COUNTRY_INFERENCE_RULES` is the one that fails CI if omitted.
