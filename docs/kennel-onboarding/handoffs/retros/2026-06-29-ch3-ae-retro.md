## Handoff GAPS → research-prompt / platform-notes improvements (the actionable part)

1. **🔑 NEW LESSON — a "Run N" archive can hide a restart-numbered sub-series.** The handoff named
   "VCH3" as one of Creek's special-run series to skip, but did NOT flag that the COVID-era **Virtual
   Creek** runs are *also* titled `Run <N>` with **low numbers 1–20** (all 2020) and live in the SAME
   `/?cat=4` category — the main series paused at #1997 and resumed at #1998, while VCH3 ran 1–20 in
   parallel. A naïve "require `Run <N>`" filter ingests them as bogus "Creek H3 Trail #1..#20". Caught at
   extraction by **sorting the frozen set by date** and spotting `Run 1 @ 2020-03-25 "Virtual Creek…"`;
   filtered cleanly by the **documented run-number floor** (`runNumber < 1000`; the real archive starts
   Run 1000 @ 2001), NOT by venue text (one legit main-series run, #2011, was *also* held virtually).
   → **Research-prompt add:** when a kennel ran a COVID/virtual series, expect restart-numbered rows in
   the main run category; give the implementer the run-number floor. Captured as memory
   `reference_runn_archive_hidden_restart_subseries`.
2. **Reject impossible title dates (Codex P2 — real correctness fix).** Trusting the title date is right,
   but `isoDate` originally accepted `2026-06-31`; merge's `parseUtcNoonDate` would have rolled it to
   Jul 1, silently moving a run. Fix: round-trip through `Date.UTC` and return `null` on overflow. →
   **Platform-note add for year-bearing-title adapters:** always validate the (y,m,d) round-trips before
   building the ISO string.
3. **Don't trust review-bot assertions about language semantics — verify.** BOTH Gemini and claude[bot]
   claimed "JS `\s` does not match ` ` (only `\p{Zs}` with `u` does)". This is **false** — JS `\s`
   matches U+00A0. A one-line `node -e '/\s/.test(" ")'` settled it (`true`); the existing `\s+`
   normalization already collapses `&nbsp;`. Two independent bots stating the same wrong fact is *not*
   corroboration. (The `/maps/i` href test likewise already covered `goo.gl/maps`, `maps.app.goo.gl`,
   `google.com/maps` — all contain "maps".)
4. **`Event.dateUtc` renders as `…T15:00:00Z`, not noon — house style, NOT a bug.** Spot-check showed
   Creek events at 15:00Z; cross-checked an existing kennel (Desert H3) → identical representation. It's
   the timestamp-without-tz storage convention, flagged in memory `reference_event_dateutc_is_timestamp_without_tz`.
   → confirm against a sibling kennel before treating a dateUtc time-component as wrong.
5. **`tsx` DB probe needs `DATABASE_URL` in env; `BACKFILL_ALLOW_SELF_SIGNED_CERT` is a no-op.** An inline
   `npx tsx -e` importing `@/lib/db` hit localhost ("server does not support SSL") until I ran
   `set -a && source .env && set +a` first — the inline script doesn't auto-load dotenv (the committed
   backfill scripts do via `import "dotenv/config"`). `BACKFILL_ALLOW_SELF_SIGNED_CERT` appears only in
   doc comments and is never read by code; the real SSL lever is `db.ts`'s `NODE_ENV` check + the
   Railway URL being present. → **platform-note correction.**
