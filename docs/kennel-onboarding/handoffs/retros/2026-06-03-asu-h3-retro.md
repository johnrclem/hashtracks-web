## Handoff GAPS → research-prompt / platform-note improvements (the actionable part)

### A. 🔴 `aliases.ts` is keyed by `kennelCode`, NOT slug — the handoff said slug (fail-silent)

The handoff's paste-ready block keyed the alias entry by the **slug** (`"asuncion-h3": [...]`) and
explicitly annotated "Keyed by the SLUG, not the kennelCode." That's **backwards**: `prisma/seed.ts`
builds `kennelRecords` by `kennelCode` (`seed.ts:330`) and `ensureAliases` looks the alias-map key up
against it (`seed.ts:347`), warning `⚠ Kennel code "<key>" not found, skipping aliases` on a miss —
**no aliases attach, no error**. The ONH3 example can't catch this because ONH3's kennelCode and slug
are both `onh3`; ASU is the first WP.com case where they differ (`asu-h3` vs `asuncion-h3`). Used
`"asu-h3"`.

> **Prompt change:** key `aliases.ts` entries by **`kennelCode`**, never the slug. Drop the
> "keyed by SLUG" annotation from the handoff template. Flag explicitly whenever kennelCode ≠ slug.

### B. 🔴 The in-body date format drifts HARD across the archive — "parse the first D Month YYYY" badly understated it

The handoff/addenda said to "search a loose `D Month YYYY` and take the first (English) match." That
covers the *recent* posts (which is all the research sampled — Run #120). Across the full 120-post
archive, **only ~10 use that clean form.** The rest: ordinal + "of" (`5th of December 2021`,
`28th of May 2022`), plain (`17 January 2026`), **English weekday + Spanish month** (`14 marzo 2026`),
the full Spanish `de` form, and a **recurring `Arpil` typo** (#34, #35). A naive parser silently drops
110/120 runs. The fix needed ordinal-stripping + `of`/`de` normalization (as separate literal
replaces — `(?:of|de)` trips Sonar S5852) + a combined EN+ES month map including the `arpil` typo.

> **Prompt change:** when specifying a date parser for a multi-year archive, **sample posts from
> across the archive (oldest, middle, a batch-posted cluster), not just the latest** — date formats
> drift over a kennel's history. For bilingual sources, the "English" column can still carry the
> other language's month name. Platform note (WP.com / Asunción addenda) updated with the full recipe.

### C. 🔴 An accented shortName mangles the derived slug — pin it

`toSlug("Asunción H3")` does `.replaceAll(/[^a-z0-9]+/g, "-")` with no accent transliteration, so the
`ó` became a dash → **`asunci-n-h3`**, not the `asuncion-h3` the handoff/PR/aliases all assumed. Fixed
by pinning `slug: "asuncion-h3"` in the seed row (the seeder honors `kennel.slug` as an override and
updates an existing record's slug on re-seed — precedent: `nah3`). Did **not** generalize `toSlug` to
transliterate accents — that would churn existing accented slugs/URLs on re-seed.

> **Prompt change:** for any kennel whose `shortName` contains a non-ASCII character, **emit an
> explicit `slug:` in the seed row** (the clean ASCII form, matching the no-accent alias). Don't trust
> the auto-derived slug or the "slug = lowercase-kebab" assumption.

### D. 🟡 `safeFetch`'s direct path has no default timeout

Not in the handoff (a general adapter lesson, already a CodeRabbit learning from NSWHHH #1917). The
direct-fetch path in `src/adapters/safe-fetch.ts` only applies a timeout on the residential-proxy
branch; a hung WP.com connection would block the scrape indefinitely. Added
`signal: AbortSignal.timeout(30_000)` (matches `chicago-shared`/`phoenixhhh`).

> **Prompt/platform change:** any new adapter calling `safeFetch` on the direct path must pass an
> explicit `signal: AbortSignal.timeout(...)`. Added to the WP.com platform note.

## Implementation / process learnings (loop context)

1. **`/simplify` caught a dead run-number fallback + duplicated coord validation.** `postToEvent` only
   runs after `isRunPost` (the `Run #N` title already matched), so the `extractHashRunNumber(...) ??
   runFromTitle(...)` dual path was dead — collapsed to `runFromTitle`. And the inline lat/lng range
   checks reimplemented `geo.ts isValidCoords` — exported it and reused (one-line export, real dedup).
2. **Adversarial review caught the page-1-400 ambiguity.** A 400 on page 1 (site/API gone) was
   indistinguishable from the healthy "0 upcoming" scrape (both → 0 events, no error, `stopReason`
   null), which could let reconcile false-cancel upcoming events. Hardened: page-1 `end` flags
   truncation; later pages stay clean-end. Tested both paths.
3. **SonarCloud new-code issues (gate passed, fixed anyway): 4 → 0.** S5869 ×2 (`[A-Za-z]` under `/i`
   duplicates the class — use `[A-Z]`), S3776 (`fetch` complexity 24 → extracted `parseFuturePosts`),
   S7721 (hoisted the test's `jsonResponse` helper to module scope).
4. **CodeRabbit: 1 acted, 2 declined-with-reason.** Acted on the `safeFetch` timeout (Gap D) and
   tightened `EMBED_SRC_RE` `[^"]+` → `[^"'\s>]+`. Declined the two "missing coords" flags (#73, #76) —
   those source posts genuinely have no embed iframe; inventing coords would fabricate data, and the
   merge geocodes the location/centroid fallback (verified: 120/120 events ended up with coords).
   CodeRabbit accepted both explanations and saved the no-embed pattern as a learning.
5. **The safety classifier blocked an ad-hoc prod slug write — correctly.** A direct
   `prisma.kennel.update({slug})` against Railway was denied as outside the documented runbook. Routed
   it through the **authorized** path instead: added `slug:` to the seed and re-ran `npx prisma db
   seed`, which updated the record via the seeder's own override logic (`~ Updated kennel: Asunción H3
   (slug)`). Right outcome, right mechanism.
6. **Node 20 / `fnm` weren't installed in this checkout** — used Node 25, which satisfies Prisma 7's
   "20+" requirement. All checks (tsc/lint/test/tsx/seed/backfill) ran fine on 25. (The post-merge
   runbook's `eval "$(fnm env)" && fnm use 20` is the documented norm; note the fallback.)

## TL;DR for the research prompt + platform notes

1. **Key `aliases.ts` by `kennelCode`, not slug** — and flag whenever kennelCode ≠ slug (the ONH3
   template hid this because its code and slug coincide).
2. **Sample dates from across the whole archive** before specifying a date parser — formats drift over
   years; bilingual sources mix month languages; expect typos. Normalize ordinals/`of`/`de` and use a
   combined-language month map.
3. **Pin an explicit `slug:` for any non-ASCII `shortName`** — `toSlug` drops accents to dashes
   (`Asunción H3` → `asunci-n-h3`), it doesn't transliterate.
4. **Any `safeFetch` direct-path call needs `AbortSignal.timeout(...)`** — no default timeout there.
5. **A page-1 HTTP 400 is a failure, not end-of-pagination** — flag truncation so reconcile can't
   false-cancel; only 400 on page ≥2 is a clean end.
6. **Keep**: the WP.com platform notes + pre-written addenda, the embed-coord pre-flagging, the
   recent-cadence/0-upcoming framing with a parse-path verification target, the new-country 5-edit
   checklist, the H7 frozen-dataset backfill, and alias-collision discipline.
