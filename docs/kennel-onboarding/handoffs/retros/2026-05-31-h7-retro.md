## Handoff GAPS → research-prompt / implementation-pattern improvements

### Gap A. 🟡 Historical archive dismissed too quickly — "too messy, defer" was inverted

The handoff called the hamburghash.blogspot.com archive "free-form prose recaps with inconsistent
date/run formats — low priority, defer." The reality: **105 structured announcement posts** (2021–
2026), 93/105 with explicit run numbers in the title and a stable labeled body (`Hare : … Where :
… When : Sunday 8 June, 2PM …`). 101 clean events were backfilled in the same PR session.

This is the same failure mode as WSH3's source inversion (confident claim with no evidence
artifact), just lower severity (a deferral rather than a fabrication).

**Process change:** before dismissing a historical source as "too messy to parse," the research
agent should pull a sample (e.g. Blogger Atom feed: `…/feeds/posts/default?alt=json&max-results=8`)
and evaluate the **actual title/body structure**. "I pulled 8 posts and they are free-form prose
with no structured fields" is acceptable; "probably too messy" without a sample is not.

**Implementation pattern confirmed:** for a ONE-SHOT read-only archive (source disabled forever
after), the right pattern is **freeze a curated `scripts/data/<k>-history.json` + dumb loader** —
not a committed parser. Use a throwaway parser as the extraction tool, curate the output (PII
scrub, sequence-gap check, hand-fix typos), then freeze. The H7 parser was initially committed,
then correctly reverted to a frozen dataset at user request. The frozen-data pattern is now
documented in `feedback_verify_handoff_backfill_feasibility`.

### Gap B. 🔴 `shortName` affects title synthesis — the handoff didn't warn Claude Code

The handoff proposed `shortName: "H7"` (≤4 chars). `friendlyKennelName` short-circuits to the
shortName directly only when it's >4 chars; otherwise it falls back to deriving from `fullName`.
H7's fullName is "Hansestadt Hamburg Hash House Harriers Hummel Hummel" — which the then-current
function left entirely verbose ("Hansestadt Hamburg Hummel Hummel H3"), because the mid-string
"Hash House Harriers" strip wasn't implemented yet. Result: 111 events synthesized with verbose
titles, requiring a follow-up PR + post-merge SQL cleanup.

The handoff should have flagged: *"shortName ≤4 chars → `friendlyKennelName` will derive from
fullName; confirm the derived output is acceptable before shipping."* A quick
`friendlyKennelName("H7", "Hansestadt Hamburg Hash House Harriers Hummel Hummel")` in a
`npx tsx -e` snippet would have caught this before any PR.

**Prompt change:** add to the "Adapter notes / pre-impl checks" section:

> **If `shortName.length ≤ 4`, verify `friendlyKennelName(shortName, fullName)` produces a
> readable title.** Run `npx tsx -e 'import {friendlyKennelName} from "./src/pipeline/merge";
> console.log(friendlyKennelName("<shortName>","<fullName>"));'` at implementation time.
> If the output is verbose or garbled, either (a) use a longer shortName (e.g. "Hamburg H7"
> instead of "H7"), or (b) open a `friendlyKennelName` fix alongside the onboarding.

**Side effect fixed:** this discovery drove PR #1895, which generalized `friendlyKennelName` to
strip mid-string "Hash House Harriers" for 5 affected kennels (not just H7). The fix benefited
the whole codebase.

### Gap C. 🟡 `foundedYear` declared "unverifiable" but was findable in the same archive

The handoff said `foundedYear` was "unverifiable — no working source." During backfill, the
Blogspot archive contained the post "30th Birthday run — Saturday June 3rd 2023 — Bremen" →
`foundedYear: 1993` (high confidence). The research agent apparently didn't look at the archive
at all (since it dismissed the archive as prose recaps).

**Process change:** when `foundedYear` is flagged unverifiable, include a note on what was
searched. If a Blogspot archive exists, check for milestone-run posts ("30th Anniversary", "Run
#500", "celebrates X years") — they're often the most reliable founding year signal.

### Gap D. 🟡 Seed update path assumed to propagate `shortName` — it didn't

The handoff (and the initial PR) changed `shortName: "H7" → "Hamburg H7"` in `kennels.ts`. Qodo
caught in PR review that the seed's **update path** only NULL-fills `PROFILE_FIELDS` — it never
updates `shortName` on an existing row. The rename would have been a no-op in prod.

This required a non-trivial `seed.ts` change (add shortName rename propagation + alias-the-old-
shortName). The blast-radius check revealed only 2 rows affected (h7, yakh3 case-fix) and that
fullName propagation would clobber 7 admin-curated rows, so fullName was explicitly excluded.

**Process change:** when changing a kennel's `shortName` in `kennels.ts`, add a note:

> ⚠️ The seed's update path does not overwrite existing `shortName` values by default. The
> `shortName` rename propagation (added in PR #1895) now handles this, but verify it's still
> active before relying on it at post-merge re-seed.

### Gap E. 🟢 (Minor) The HC source test count guard wasn't updated

The `sources.test.ts` regression guard (`expect(hc.length).toBeGreaterThanOrEqual(3)`) was stale
after adding the 4th HC source. The Claude bot caught it in review. Should have been caught
earlier by the implementation agent or `/simplify`.

**Process change (already in place):** the Claude bot's review catches this class of stale guard.
The `/simplify` skill found no issue because the guard still *passed* (4 ≥ 3). Low priority — no
prompt change needed.

## Frozen dataset validation checklist (new — from backfill experience)

For any one-shot frozen `scripts/data/<k>-history.json`:

1. **PII scrub** — grep for `@` (emails) and phone patterns.
2. **Run-number sequence continuity** — numbered runs should be ~14 days apart per step; flag
   `gap/step < 8.5d` or `> 40d` as potential date misparses.
3. **Gap-sanity check** — event date within −1…+60d of the post's published date; catches body-
   vs-title typos (the #690 2025-09-14 vs Sep-21 bug).
4. **Monotonicity** — run numbers should not decrease.
5. **Field bleed** — hares/locations containing `:` characters may indicate a label boundary was
   missed; inspect the raw text.
