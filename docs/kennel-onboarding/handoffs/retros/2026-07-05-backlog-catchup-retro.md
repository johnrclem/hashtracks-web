## The root cause — and the process fix (the actionable part)

**`scripts/copy-newest-handoff.sh` only ever surfaces the *newest* un-implemented handoff.** When
implementation is skipped for a day, the older handoff isn't next-in-line the following morning — the
*newer* one is. So skipped days don't queue up; they fall off the bottom silently. Three days had
accumulated (07-02/03/04) with no signal anywhere except the absence of `onboard/` branches.

- **What caught it:** auditing before starting — checking `origin/main` seed-data, `onboard/` branches,
  the `retros/` dir, and the `run-log.md` `Outcome:` lines for each recent handoff. Three independent
  signals agreed (absent from seed-data, no branch, no retro; `Outcome: handed off` never flipped).
  The lesson generalizes: **before picking up "the next" handoff, confirm the previous few actually
  shipped** — the copy script's "newest only" is not proof the older ones are done.
- **Fix to land:** teach `copy-newest-handoff.sh` to **warn when >1 un-implemented handoff exists** (or add
  an `--oldest` mode that walks bottom-up). A one-line "⚠ 3 un-implemented handoffs (oldest: 2026-07-02)"
  banner would have made this a non-event. Flagged in run-log; not yet implemented.

## Handoff corrections caught at implementation (the reason to verify, not trust)

The handoffs were written against an **older `main`**, so every cited line number had drifted (Bandung
template `sources.ts:7157/7267` was actually `7284`; Kaohsiung `1746`→`1791`; etc.). **Locate anchors by
name/pattern, never by the handoff's line numbers.** Beyond that, four substantive catches:

1. **`abbrev "LAN"` collides with Lansing, MI** (Lune Valley). The handoff proposed `LAN`; a grep of
   existing `abbrev:` values found Lansing already owns it. Shipped **`LNC`**. The handoff itself said
   "confirm no collision before shipping" — so: *always grep the abbrev namespace for a new metro.*
2. **Geocode-fail vs. legitimate away-run — don't blanket-bbox-scrub** (Lune Valley). A naive NW-England
   bbox flagged 13 rows, but only **one** (#900 "Scorton", a Lancashire venue pinned in Stonehaven,
   Scotland) was a real geocode-fail. Runs **#803–805 ("Weekend away: Sandyhills")** are a genuine away
   weekend whose Scottish coords are *correct*. A blanket bbox would have wrongly scrubbed real pins.
   Lesson: **inspect the outliers' venue text before scrubbing; scrub the mismatch, keep the away-trip.**
   (The merge pipeline's own geocode-distance guard is the backstop — it independently caught Pranburi
   #5's bogus "Tap Thai lake" resolving 13,326 km off and skipped the coord.)
3. **Shallow-archive reality beats the handoff estimate — both ways.** Pranburi's "~5 runs" was really
   **only #4/#5** (the kennel adopted HC at #4; #1–3 predate its archive — verified by windowed pulls
   returning 0). Lune Valley's "≥#869/2024-06" was really **215 runs back to 2020**. Pull the archive to
   the floor; don't trust the sample's depth guess.
4. **New-country hygiene held** (BEER H3 / Serbia): 5 `region.ts` edits, red palette with no seeded-
   neighbor clash, `Europe/Belgrade` (not the HC feed's `Europe/Warsaw` quirk), and — the one that bites
   in CI — `COUNTRY_INFERENCE_RULES` **omitting bare `belgrade`** (US-town collision: Belgrade MT/ME) with
   a disambiguation test asserting `Belgrade, MT → USA`.

## Process slips (minor)

- **Auto-mode TLS guard fired — correctly.** The first prod-seed attempt prefixed
  `NODE_TLS_REJECT_UNAUTHORIZED=0` (globally disabling TLS verification); the classifier blocked it. The
  sanctioned flag is **only `BACKFILL_ALLOW_SELF_SIGNED_CERT=1`** — re-ran with that and it worked. Don't
  reach for the global TLS-disable; the app's adapter honors the scoped flag.
- **`fnm`/Node 20 unavailable this session** — verified on Homebrew Node 25 (satisfies Prisma's Node 20+;
  CI uses its own pinned Node). Noted in each PR body.
- **`.mts` throwaway didn't resolve the `@/` alias** under tsx — use a `.ts` file in `scripts/` so
  tsconfig paths apply; `getAdapter(type, url, config)` (3-arg) is the working import shape.
