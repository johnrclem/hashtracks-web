## What needed correction (fix these in the research prompt)
1. **`aliases.ts` format was wrong.** Handoff wrote an array of `{ kennelCode, alias }` objects; the file is actually `Record<string, string[]>` (`"zh3": ["ZH3", ...]`). → *Have research read the current seed files and emit blocks in their real shape, not an assumed one.* **Highest-impact fix** — it's a guaranteed manual correction every run otherwise.
2. **Region color pairing didn't match codebase convention.** Handoff used `bg-red-100 / #dc2626` for **both** country and metro. Convention is **darker country (`-200`) / lighter metro (`-100`)**, and the two shouldn't share a pin color. → *Tell research to mirror a sibling COUNTRY+METRO pair (e.g. Germany/Berlin) for the shade/pin pattern.*
3. **Too many region aliases, with near-duplicates.** Handoff gave 4 Zürich aliases incl. `"Zurich, CH"` + `"Zürich, CH"`; sibling metros carry 1–2 (name + native spelling). A reviewer (gemini) flagged the dup; trimmed to `["Zurich", "Zürich, Switzerland"]`. → *Match sibling alias density; drop `", <code>"` forms.*
4. **Trailing-zero numeric literal.** `centroidLat: 46.80` tripped SonarCloud's "unnecessary trailing zero." → *Emit clean literals (`46.8`).*
5. **Flat schedule fields lacked the "fallback" comment.** The current convention (lbh3, PR #1684) keeps `scheduleDayOfWeek`/`scheduleTime` alongside `scheduleRules` **with a comment** that scheduleRules are authoritative. Handoff included the flat fields but not the comment. → *Include the one-line comment, or instruct the implementer to.*
6. **Verification vs. prod-seed conflation.** The directive said "`npx prisma db seed`, then live-verify." In practice live-verification is best done by **running the adapter directly (no DB write)**; the prod seed + scrape is a **separate post-merge step**. → *Split these explicitly: (a) adapter-only live-verify pre-PR; (b) post-merge `db seed` + scrape.*

## Net
Items 2–5 were caught by `/simplify` + bot review and cost minutes. **Item 1 (aliases format) is the one worth fixing first** — a deterministic miss the research prompt can eliminate by grounding its seed blocks in the actual file shapes. Everything under "what worked well" is load-bearing and should be kept.
