## What needed correction (fix these in the research prompt)
1. **🔁 `aliases.ts` format wrong — AGAIN (recurred from ZH3 retro item 1).** §8d emitted an array of `{ kennelCode, alias }` objects; the file is `Record<string, string[]>` (`"bali-hash-2": ["Bali Hash 2", ...]`). This is the **same deterministic miss as run #4 ZH3**, so the fix from that retro **either wasn't applied to the prompt yet or didn't take**. → **Highest priority.** Have research actually read `prisma/seed-data/aliases.ts` and emit the block in its real shape. A recurring guaranteed-correction is the clearest prompt-ROI there is.
2. **🆕 `config.upcomingOnly: true` was MISSING from the source block — a real correctness bug, caught by Codex (P2).** Because the adapter only emits the home page's ~12–22 most-recent posts and this is the kennel's **sole source**, the reconciler (default `upcomingOnly=false`, 90-day window) would mark a past run `CANCELLED` once it ages off the listing — silent data loss. The research system's *own* ONH3 source block (same shape: blog, recent-only, sole source) already has this guard, so research had the precedent and didn't apply it here. → **Add a prompt checklist rule:** *if the source is a recent-only listing (blog home page / "upcoming runs" widget) and is the sole source for the kennel, the source block MUST include `config: { upcomingOnly: true }` with a comment.* High value — this was the only true bug in the handoff.
3. **🆕 "Archive depth" was overstated ~22×.** The handoff claimed "~1,747 runs of archive… deep archive available" and recommended a full-archive backfill. Reality: the Ghost blog only publishes `next-run-map` posts back to **run #1660 (Sept 2024) — ~78 posts**; pagination ended after ~10–13 pages. The "1,747" was the **current run *number***, not the available archive depth — the two were conflated. The backfill still worked (captured all 78), but the effort estimate and "one-shot ~1,747-run backfill" framing were wrong. → **Tell research to distinguish "latest run number" from "verified paginated archive depth": actually walk 2–3 `/page/N` pages and report the oldest post's date/run#, not infer depth from the run number.**
4. **🔁 Region color was a flagged guess (related to ZH3 items 2–3).** §8a.1 used a rose palette **and openly admitted** "current values are a guess and likely collide with an existing Asia kennel." Improvement over ZH3 (it flagged the risk) but still shipped a guess; I switched to teal (distinct from red Japan/Singapore, orange Thailand) with darker-country/lighter-metro shades. → *Don't emit a self-admitted guess — have research scan existing same-continent regions, pick a free palette, and apply the `-200` country / `-100` metro + distinct-pin convention (mirror a sibling COUNTRY+METRO pair).*
5. **🔁 Verification vs. prod-seed split (recurred from ZH3 item 6, minor).** §10's live-verify checklist was correctly adapter-only, but the framing still blurs pre-PR live-verify vs. the separate post-merge `db seed` + scrape. The README now documents the split; keep the prompt language aligned with it. *(Note: this run also exposed a stale doc — `hashtracks.com` in CLAUDE.md is a parked page; real prod is `hashtracks.xyz`. Unrelated to research, flagged separately.)*
