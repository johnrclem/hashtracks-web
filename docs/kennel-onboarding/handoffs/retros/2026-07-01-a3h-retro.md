## Handoff GAPS → platform-notes improvements (the actionable part)

1. **🔑 NEW LESSON — a "duplicate run number" can be a CANCELLED slot, not a second trail.** The handoff
   (from its sample) said #2167 appeared on 2025-12-28 AND 2026-01-11 → "keep both, dedup by date." The
   ground-truth pull showed the 2025-12-28 row was `EventName:"HASH CANCELLED"`, `IsCountedRun=0`, venue
   `"Tbc"` — a scrubbed slot, not a trail. **Dropped it** (ingesting it would show a "HASH CANCELLED" past
   card), which also resolved the apparent dedup collision for free, leaving one clean #2167. → **platform-note:**
   check each dup row's `EventName`/`IsCountedRun` before assuming both are real; filter `IsCountedRun=0` +
   a `HASH CANCELLED` name guard in the extractor.
2. **🔑 NEW LESSON — HC `EventDescription` frequently carries PII; NEVER copy it to `description`.** The
   handoff's field-fill table said "description 0/28 (EventDescription empty)." It was the opposite — the
   field was FULL, of hare **emails and phone numbers**. The live adapter emits no `description` at all, so
   omitting it is both faithful and PII-safe. → **platform-note** (+ reinforces the existing "backfill omits
   cost/description" corollary with the PII *reason*).
3. **Venue-as-coords-text is normal for A3H and correctly yields an empty `location`.** Most rows stored
   `LocationOneLineDesc` = `"37.1, -8.0"` (raw coords in the venue field); `composeHcLocation`'s coords-only
   drop leaves `location` empty while the real `lat/lng` (19/23) pin the event and the village survives in
   the title. Not a parsing miss — expected, and matches live-adapter output. → **platform-note** (documents
   the "empty location is correct here" case so a future reviewer doesn't try to "fix" it).
4. **⚠️ PROCESS SLIP — Edit/Write landed on the MAIN-repo copies, not the worktree.** The exploration
   agents reported main-repo absolute paths (`/…/hashtracks-web/prisma/seed-data/…`), and the seed/region
   edits followed those paths → they modified the main repo (on `main`), while only the `cd`-into-worktree
   Bash commands (logo download, JSON copy) landed correctly. Caught immediately by a `git -C <worktree>
   status` showing the tracked files as *unmodified* → `cp`'d the changes main→worktree, `git restore`'d
   main to clean, verified both trees. → **reinforces [worktree-bash-cwd-resets-to-main]:** always target
   the worktree-prefixed path for Edit/Write, and `git status` BOTH trees before committing.
5. **Gemini caught a real accented-`\b` inference gap.** The ASCII group `\b…|loule|…\b` can't match
   "loulé" (the literal ends in `e`; `é` is a non-word char), so the Loulé spelling in the backfill titles
   wouldn't infer Portugal. Fixed with a bare `[/loulé/, "Portugal"]` rule (matching the existing
   `københavn`/`göteborg`/`malmö` accented-token style; input is lowercased before matching → no `i` flag).
   Seed kennel carries explicit `country`, so this only hardens the research path — but it's a clean,
   correct catch worth applying.
