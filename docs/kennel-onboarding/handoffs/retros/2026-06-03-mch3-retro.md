## Handoff GAPS → research-prompt / platform-note improvements (the actionable part)

### A. 🔴 The handoff said "leave `title` undefined → merge synthesizes Trail #N" — but `MeetupAdapter` SETS the title

This is the big one, and it's a **prompt-level** misconception, not a one-off. The handoff
(following daily-prompt line ~679: *"`title` — leave `undefined`; `merge.ts` synthesizes
`<KennelName> Trail #N`"*) instructed the backfill to leave `title` undefined so merge would
synthesize `"Mexico City H3 Trail #N"`. **Live verification disproved this for MEETUP:**
`MeetupAdapter.buildRawEventFromApollo` sets `title: cleanMeetupTitle(ev.title)` — the real Meetup
event name — for **every** Meetup source, and `merge.ts` (the `sanitizeTitle` branch) **keeps an
adapter-provided title**, only synthesizing `Trail #N` when title is *absent*. There is no
`MeetupConfig` knob to suppress it.

Consequence if you follow the generic rule: live-scraped runs render as `"HASH #756- SPRING HASH !"`
while the backfilled past runs render as `"Mexico City H3 Trail #756"` — **mixed titles on one
kennel, plus title churn** on the in-window overlap (#754–#756, which the live adapter re-scrapes
within `scrapeDays`). Fix applied: capture the real cleaned Meetup titles from a live
`adapter.fetch(source, {days})` against the `?type=past` page and **freeze them in the backfill
JSON**, so backfilled history matches what the adapter stores going forward. This still avoids the
*real* anti-pattern (a bare theme like "SPRING HASH" as the title) because these are full event
names.

> **Prompt change (done):** the "`title` — leave undefined" rule now carries a **MEETUP carve-out**:
> the adapter always sets the cleaned Meetup name and merge keeps it, so (a) don't instruct
> "leave title undefined" for Meetup, and (b) Meetup backfills must **freeze the real cleaned Meetup
> titles** (capture via a live `?type=past` fetch), never leave them undefined.
> **Platform note (done):** added to the Meetup section of `source-platform-notes.md`.
> Also captured in agent memory `reference_meetup_adapter_sets_title`.

### B. 🟡 `scheduleTime` "14:00" again — and a backwards gather-vs-start time

Recurring from Paris Gap E: the handoff's seed used `scheduleTime: "14:00"` (24-hour); the repo
convention is 12-hour `"2:00 PM"`. **New twist:** the handoff's "2:00 PM gather" (from Mexico News
Daily editorial) actually *contradicted* the consistent Meetup start of **13:30** — you can't start
the trail at 1:30 if you gather at 2:00, so the MND figure is rounding. Set `scheduleTime: "1:30 PM"`
to match the operational Meetup start, not the editorial gather time.

> **Prompt change (done):** reinforced 12-hour `"H:MM AM/PM"` and added: when an editorial "gather
> at ~X" disagrees with the source's actual event start times, **seed the operational start time**
> (what the events list shows), not the prose gather time.

### C. 🟡 `scheduleFrequency` stated "Biweekly" vs observed ~monthly

The kennel self-describes as "every other Saturday" but the observed Meetup cadence over the last 6
months was ~monthly. Kept the kennel's stated `Biweekly` but added a `scheduleNotes` capturing the
observed reality and the occasional earlier away-trip starts (the CI Claude reviewer's suggested
minimum). Good middle ground — honors the kennel's own copy while being honest on the page.

> **Prompt change (done):** when stated cadence and observed cadence disagree, keep the stated
> `scheduleFrequency` but record the observed reality in `scheduleNotes` (don't silently mislabel).

### D. 🟡 Inference-regex `\b` consistency on the multi-alternative rule

The Mexico rule shipped as `/\bm[eé]xico\b|\bcdmx\b|ciudad de m[eé]xico/` — the third alternative
lacked the `\b` the other two had. Gemini + the in-CI Claude reviewer flagged the inconsistency;
fixed to `…|\bciudad de m[eé]xico\b/`. (Accidental match was near-impossible in practice, but the
inconsistency is a smell.) Relates to Paris Gap B (non-ASCII `\b`) — here it was ASCII, just
asymmetric.

> **Prompt change (done):** when an inference rule has multiple `|` alternatives, apply `\b`
> consistently to all of them.

## Implementation / process learnings (loop context)

1. **🔴 Worktree path gotcha bit hard (`feedback_worktree_bash_cwd_resets_to_main`).** The first
   round of Edits used main-repo absolute paths (`/Users/johnclem/Developer/hashtracks-web/…`) and
   landed in the **main checkout — which was on an unrelated `docs/asu-h3-retro` branch** — while
   Bash (cwd = worktree) read the worktree's untouched files, so `tsc`/grep saw none of the changes.
   Caught it when `inferCountry` returned all-USA. Reverted only the 4 mis-placed files in main
   (left that branch's own uncommitted docs changes intact) and re-applied everything to the
   worktree with worktree-prefixed paths. **Always use worktree-prefixed absolute paths in a
   worktree session;** verify a change actually landed (grep the worktree copy) before debugging
   "why didn't it take effect."
2. **`/codex:adversarial-review` is disabled for automated (Skill-tool) invocation in this harness**
   — only the user can type it. Ran `/simplify` (4 agents, clean) and flagged the codex step for the
   user. Don't burn cycles trying to auto-invoke it; note it for manual run.
3. **Post-merge ran from the worktree, not the main checkout** — the main checkout was on another
   branch (`docs/asu-h3-retro`) with its own WIP, so switching it was off-limits. Verified the
   worktree's seed-data/region files were **byte-identical to `origin/main`** (empty `git diff
   origin/main`) before seeding, then sourced the prod `DATABASE_URL` from the main `.env` and ran
   `prisma db seed` + `BACKFILL_APPLY=1 …` from the worktree. Clean (Created 1 / Updated 372;
   backfill 10 created). The byte-identical check is the guard against the
   `feedback_concurrent_seed_reverts_source_config` hazard.
4. **`/simplify` flagged the `FrozenRun`/`toRawEvent` mapping layer** (asu-h3 stores `RawEventData`
   directly in JSON). Kept it — it sets the constant fields (`kennelTags`, `sourceUrl`) once in code
   instead of repeating them per row and centralizes the `null→undefined` tri-state conversion.
   Defensible deviation, noted as a skip.
5. **Quality gates clean first try** — Sonar 0 issues / 0 hotspots, Codacy 0, CodeRabbit no
   actionable, Codex no major issues. The real catches were the bots' regex-`\b` + schedule notes
   (above) and `/simplify`'s mapping-layer note.
