## What the handoff got RIGHT (keep doing)

1. **The "UNVERIFIED DOM" honesty + "capture real markup FIRST" directive.** Chrome MCP auto-denied
   the brand-new `vh3.ca` domain during research, so the handoff could only describe the structure
   from *rendered text* and said so loudly — making "fetch the real DOM and build the fixture from it"
   step 1 of implementation. The described shape (rich top cards + clean bottom schedule lists, match
   by run #) turned out **exactly correct**; only the wrapping markup needed discovery. That framing
   is the right way to ship a handoff against an un-inspectable new domain.
2. **Two-source-cities-in-one-queue-row catch.** The queue row was labelled "VH3 / **Vancouver**" but
   cited `vh3.ca` (**Victoria**, Vancouver Island — a different city). The handoff caught it, onboarded
   Victoria, and split the dead `vanhash.com` (real Vancouver, "2019 Hareline") out as its own
   `blocked: stale` row. Don't let a queue label override the source's actual identity.
3. **Card-first precedence for last-minute changes.** The top "Up Cumming" cards are the curated
   near-term view; the bottom schedule list is the season backbone. The handoff specified card
   date/time wins over the schedule list, so a last-minute time change on the card beats the stale
   bottom-list time. Implemented as `card?.date?.startTime ?? sched?.startTime` and a card-first `pd`.
4. **A concrete verbatim sample corpus** (all three kennels' run lines, the `#944 (2027)` form, the K9
   Boxing Day card) gave an exact live-verify + fixture oracle.
5. **kennelCode collision discipline.** `vh3` / `dsmh3` / `vk9h3` all grep'd clear against the many
   `*vh3*` codes (mvh3, svh3, hvh3, …); bare "VH3"/"K9 H3" aliases scoped per-code. No collision.

## Handoff GAPS → research-prompt / platform-note improvements (the actionable part)

### A. 🔴 New STATE_PROVINCE region needs a `seed.ts` `stateMetroLinks` entry — a 5th edit in a different file

The handoff's region section listed "British Columbia province + Victoria, BC metro (4 region-map
edits in `region.ts`)." All four `region.ts` edits were correct — but the **admin region hierarchy
parent-child link lives in `prisma/seed.ts`'s `stateMetroLinks` map**, not `region.ts`. Without
`"British Columbia": ["Victoria, BC"]` there, Victoria, BC seeds as an orphan METRO and never appears
under the BC province node in the admin UI. Codex flagged it P2; added in the same PR. Easy to miss
because Canada already had `Alberta`/`Ontario`/`Quebec` entries, so the *country* looked "done."

> **Prompt change (recommend):** the new-region checklist should be **"region.ts 4-edit + seed.ts
> `stateMetroLinks`"** whenever the new region is STATE_PROVINCE-level with metros under it (US states,
> CA provinces, AU states, etc.). Flat COUNTRY→METRO countries with no state tier don't need it. Phrase
> it as "add the `<Province>`: [`<Metro>`] row to `stateMetroLinks` or the metro won't parent in admin."

### B. 🔴 Gamma's whole-document `.text()` STACK-OVERFLOWS — unpredictable from rendered text

The single biggest implementation surprise, and impossible to foresee from the research run's rendered
text. Gamma nests each block in a deep div tree; `$.text()` on the root (or `stripHtmlTags(html)`)
blows the domutils `textContent` recursion stack on the real ~833 KB page. `cheerio.load` is fine —
only whole-tree extraction overflows. Fix: select the **leaf node-view wrappers**
(`[data-node-view-content-inner="paragraph"]` / `heading` / `title`) and read each shallow `.text()`.

> **Platform note (done):** the Gamma section in `source-platform-notes.md` was rewritten — the
> `⚠️ UNVERIFIED DOM` caveat is replaced with the **✅ CONFIRMED DOM** (node-view wrappers, separate
> `maps.app.goo.gl` href map) and the stack-overflow → leaf-selection gotcha is now the headline
> bullet. Also captured in agent memory `reference_gamma_platform_adapter`.

### C. 🟡 Schedule-vs-card discriminator: weekday prefix, NOT month substring

First instinct (and an early draft) keyed the schedule-list-vs-card decision on a month-name substring
in the run-line remainder. A card heading theme like **"May Day Madness"** contains "May" and would
misclassify as a schedule (dated) row. Codex caught it; switched to a weekday-prefix test
(`/^:?\s*(?:mon|tue|wed|thu|fri|sat|sun)/i`) on the remainder — schedule lines start their remainder
with a weekday, card headings never do. Immune to month-in-theme.

> **Platform note (done):** added to the Gamma section. General lesson for any multi-shape page:
> discriminate on a **structural token that can't appear in free-text** (a weekday prefix), not a
> substring that a human-written theme might contain.

## Implementation / process learnings (loop context)

1. **🔴 SonarCloud S5852 whack-a-mole — go procedural for ANY `\s*`-adjacent alternation from the
   start.** Three rounds of the quality gate failed on regex shapes, each a separate `\s*`+alternation:
   `WRITEUP_RE`'s `[^(]+\(` (negated class before that char) → rewrote to a `Hash #NNN` prefix +
   `indexOf("(")`; a `/,?\s*&\s*/g` hare-conjunction normalizer → procedural `split("&").flatMap(split(","))`.
   The trap: **fixing the hares bug (below) with a `.replace(/…hares?\s+(?:needed|wanted)/gi, "")`
   REINTRODUCED the exact shape** on a fresh line, failing the gate again. Final fix: a token filter
   (split → drop placeholder tokens via `haresNeededIn` + an anchored `NEEDED_ONLY_RE`), zero
   ReDoS-shape regex. Lesson (matches `feedback_sonar_s5852_procedural_over_regex` /
   `feedback_codacy_non_literal_regexp`): when a cleanup would put `\s*` next to an alternation, reach
   for string ops *first* — don't fix one flagged regex with another regex.
2. **🔴 Real bug caught in CI review: hares partial-loss.** `Hare: Goes Down Well & Hares needed`
   originally hit `if (!value || haresNeededIn(value)) card.haresNeeded = true; else card.hares = …`
   — the `else` never fired, so the **real hare "Goes Down Well" was discarded** and only the
   placeholder flag survived. Fix: separate the `haresNeeded` flag from name extraction; split into
   tokens, drop the placeholder token, keep the rest. A genuine data-loss bug, not a style nit — the
   `||`-collapses-two-concerns shape is worth watching for.
3. **Quality gate can pass with issues still open — clean them anyway.** After the hotspots cleared,
   SonarCloud reported the gate **passed** but listed 5 non-blocking issues (1 CRITICAL S3776
   cognitive-complexity 31 on `applyCardLine`, plus S2310 loop-counter reassignment, S107 8-param
   `buildEvent`, 2× S7735 negated conditions). Fixed all five: extracted per-label helpers
   (`applyWhereValue`/`applyHareValue`/`applyCardDateLine`) + a thin dispatch, `parseCards` →
   while-loop, bundled the three lookup maps into a `ParsedMaps` object, inverted the negated
   `explicitYear != null`. A big if/else-if label dispatcher is the classic S3776 source — plan the
   per-branch helpers up front next time.
4. **Per-kennel base year deferred with rationale (not a miss).** A reviewer suggested per-kennel base
   years; the live page uses one shared "2026 Runs" header for all three sections, so a single
   `baseYear` + per-kennel `YearState` (Dec→Jan rollover within each sorted run list) is correct
   today. Documented the deferral on the PR thread rather than over-building. Revisit only if a live
   Gamma page ever shows divergent per-kennel years.
5. **Post-merge ran from the worktree (main checkout on a docs branch).** The main checkout was on
   `docs/kennel-onboarding-audit-fixes-…` with its own WIP, so it was off-limits. Verified the VH3
   files were byte-identical to `origin/main` (the PR landed as a **true merge commit**, not a squash,
   so all three fix commits were intact), then sourced the prod `DATABASE_URL` from the main `.env`
   and ran `prisma db seed` + a forced `scrapeSource` from the worktree. The IndexNow-ping +
   `revalidateTag` warnings (`after`/static-store outside a request scope) are expected in a CLI run
   and don't affect the data. Same guard as the `feedback_concurrent_seed_reverts_source_config` /
   worktree-path hazards.
6. **`dogFriendly` plumbed end-to-end (user-approved).** Added `dogFriendly?: boolean` to `KennelSeed`
   + `"dogFriendly"` to the `PROFILE_FIELDS` whitelist in `seed.ts`; set on `vh3` and `vk9h3`. A new
   profile field needs the interface + the seeder whitelist, or the seed silently drops it.

## TL;DR for the research prompt + platform notes

1. **Gamma: whole-document `.text()`/`stripHtmlTags` STACK-OVERFLOWS** — select the
   `[data-node-view-content-inner="paragraph|heading|title"]` leaves and read each shallow `.text()`.
   (Platform note rewritten with the confirmed DOM + memory `reference_gamma_platform_adapter`.)
2. **A new STATE_PROVINCE region needs `seed.ts` `stateMetroLinks` too** (`"<Province>": ["<Metro>"]`),
   not just the 4 `region.ts` edits — else the metro orphans out of the admin province hierarchy.
   Make the new-region checklist "region.ts 4-edit **+ seed.ts stateMetroLinks**" for state/province tiers.
3. **Go procedural (string ops) for any hares/title cleanup** that would put `\s*` next to an
   alternation — S5852 flags it even when linear, and fixing one flagged regex with another regex
   re-triggers it. Token-filter, don't `.replace()`.
4. **Discriminate multi-shape lines on a structural token, not a substring** — weekday-prefix beats
   month-name for schedule-vs-card (a theme can contain a month name).
5. **Multi-kennel pages get a per-kennel fail-loud zero guard** (one error per expected tag) so a
   partial parse can't false-cancel a dropped kennel's future runs.
6. **Keep**: the `▶ FOR CLAUDE CODE` directive, the "UNVERIFIED DOM → capture real markup first"
   framing for new domains, the new-region inference-ordering/collision guard, magic-byte logo `<ext>`
   confirmation, card-first precedence, the split adapter-verify / post-merge-seed runbook,
   `aliases.ts` keyed by kennelCode, and `upcomingOnly` for rolling current-season pages.
