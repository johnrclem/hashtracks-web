## Handoff GAPS → research-prompt / platform-note improvements (the actionable part)

### A. 🟢 The DOM was CLEANER than the handoff assumed — `web_fetch` flattens labels, the real markup is semantically classed

The handoff warned the labels were a "flattened blob" (`Title:`/`Meeting Time:`/… ran together in the
`web_fetch` view) and recommended a flattened-text label scanner. The **real `index.php`** is cleanly
classed:

```html
<div class="event">
  <div class="number">2897</div>
  <div class="title">Anti-Celibacy Day</div>
  <div class="section">
    <div class="label_value"><div class="label">Meeting Time:</div><div class="value">2026/06/13 16:00</div></div>
    …Location / Geo Coordinates / Hares / Apres Trail / Hash Cash…
  </div>
</div>
```

The flattening was a **`web_fetch` rendering artifact, not the DOM.** Building the fixture from a real
`curl` capture (per the handoff's own capture-the-real-DOM mandate) surfaced the semantic classes, and the
parser became a clean `.label_value` → `Map<label,value>` (read run#/title from `.number`/`.title`).

> **Prompt/platform change:** for any PHP-SSR / template-built site, **capture the raw markup before
> choosing a parser** — `web_fetch`'s text render collapses labeled `<div>`/`<td>` pairs onto one line and
> makes a clean DOM look like a flattened blob. The handoff should say "*illustrative; confirm the real DOM
> — it may be cleanly classed*," and prefer a `.label`/`.value` (or `<th>`/`<td>`) **`Map` keyed on visible
> label text** (`Map.get`, not `obj[key]` — dodges Codacy object-injection) over a fragile text scanner.

### B. 🔴 `Kennel.scheduleTime` is 12-hour "H:MM AM/PM", NOT 24-hour — the handoff's `"16:00"` would have been the only 24-hour value in the seed

The Ready-to-paste seed shipped `scheduleTime: "16:00"`. Every one of ~470 `scheduleTime` values in
`kennels.ts` is **12-hour `"H:MM AM/PM"`** (`"4:00 PM"`, `"6:30 PM"`, …) — `"16:00"` would have been the
sole 24-hour value and broken the display formatter. Corrected to `"4:00 PM"`.

> **🔴 Two different fields, two different formats — never conflate them:** `Kennel.scheduleTime` (seed
> profile) is **12-hour `"4:00 PM"`**; the adapter's `RawEventData.startTime` (per-event) is **24-hour
> `"HH:MM"`** (`"16:00"`). The handoff put `"16:00"` in the seed by analogy with the adapter field. Same
> gotcha bit Paris/mch3 — promote it in the prompt's "Ready-to-paste seed" checklist.

### C. 🔴 `sanitizeHares` does NOT strip mid-string phones — and Korean mobiles appear in BOTH `010-…` and `+82 10-…` forms (Codex caught the `+82` miss)

The archive embeds hare phone numbers inline (`"EM Blank Space +82 10-7152-6362, EM Seoul Ultraman"`,
`"ASBO 010-2354-1741"`). The merge pipeline's `sanitizeHares` only strips **trailing** logistics/boilerplate
— it leaves mid-string phones untouched — so the backfill had to scrub them itself.

- The first scrub matched only the **domestic** `010-XXXX-XXXX` form. The **Codex adversarial review**
  caught that the **international `+82 10-XXXX-XXXX`** form (the leading 0 drops) was still leaking into the
  committed dataset — a real privacy bug that would have persisted phone numbers to public canonical events.
- A second subtlety: **inconsistent spacing** (`+82  10-…` with a *double* space) defeated a `[-\s]?`
  separator; the patterns needed `[-\s]*` / bounded `*`.
- The merge sanitizer not stripping live phones means the **live adapter** must scrub too — not just the
  backfill — so the daily scraper can't reintroduce PII a future run lists.

> **Prompt/platform change:** when an archive/source lists hares with phone numbers, **scrub PII at freeze
> time AND in the live adapter, and guard the committed dataset with a regression test that uses the same
> patterns.** Match **both** `01x-XXXX-XXXX` and `+82 1X-XXXX-XXXX` (anchor on `01x`/`+82` so year ranges
> like `1995-1996` and "Line 1 (10-15 min walk)" survive); tolerate multi-space separators. Put the scrubber
> in `src/` (not `scripts/`) so the adapter and the backfill share it. Drop bulky recap `description` from
> historical rows (where emails also hide) to cut PII surface + JSON bloat.

### D. 🟡 New regex-heavy modules trip a stack of Sonar/Codacy rules — author them clean from the start

The `sh3-pii` scrubber drew, across review rounds: **S5852** (ReDoS hotspots), **S5843** (regex complexity
> 20), **S6353** (`[0-9]` → `\d`), **S6594** (`.match()` → `.exec()`), and Codacy's **non-literal-regexp**
(on `new RegExp(str)`). The clean shape that satisfies all of them:

> - **Regex *literals*, never `new RegExp(stringConstant)`** (Codacy flags the latter even for constants).
> - **One regex per type, not one big alternation** — keeps each under the S5843 complexity budget (the
>   combined email|intl|domestic pattern was complexity 39).
> - **`\d` not `[0-9]`** (S6353); **`RegExp.exec()` not `String.match()`** (S6594).
> - **Backtracking-safe email** `[\w.+-]+@[\w-]+(?:\.[\w-]+)+` (the label class excludes `.`).
> - **String ops over `\s*`-heavy cleanup regexes** (`.split(/[,;]/).map(collapse).join(", ")`), and
>   `String.search` (ignores the global flag's `lastIndex`) for detection.
> - A genuinely-linear pattern that S5852 still flags → **mark the hotspot SAFE via the SonarCloud API**
>   (every `*`/`{n}` is over a class followed by a disjoint required token → no catastrophic backtracking).

### E. 🟡 Alias collision check missed `SHHH` — grep EVERY bare initialism, not just the obvious one

The handoff correctly omitted bare **`SH3`** (Summit/Salem/Seattle) but the proposed alias list still
included bare **`SHHH`**, which collides with **`sech3`** (Secession) and **`sgharriets`** (Singapore
Harriets). CodeRabbit + the `claude` review bot caught it. Removed; the four city/name aliases are
sufficient discriminators.

> **Prompt change:** the collision check must grep **every** bare initialism in the proposed alias list
> (`SH3` *and* `SHHH`), not just the kennelCode-shaped one.

### F. 🟢 An Explore sub-agent mis-reported a `region.ts` fact — verify sub-agent claims against the code

A research sub-agent claimed Philippines/Manila had **no `STATE_GROUP_MAP` entry** (the "city-state
pattern"). The actual code (`region.ts:3705`, `"Manila": "Philippines"`) had it — the handoff's 5-edit plan
was right. Trusting the sub-agent would have dropped a required edit.

> **Process note:** treat Explore/research sub-agent summaries as leads, not authority — confirm any
> "file X does/doesn't contain Y" claim against the live code before acting (the live repo is the authority,
> echoing the handoff's own `⚠️ verify before writing real code`).

## TL;DR for the research prompt + platform notes

1. **PHP-SSR / template sites are often cleanly semantically-classed** — `web_fetch`'s text render flattens
   labeled `<div>`/`<td>` pairs into a blob. **Capture the raw DOM first**, parse via a `.label`/`.value`
   (or `<th>`/`<td>`) **`Map.get`** keyed on visible label text — not a fragile flattened-text scanner.
2. **`Kennel.scheduleTime` = 12-hour `"4:00 PM"`** (seed profile); **`RawEventData.startTime` = 24-hour
   `"HH:MM"`** (per-event). Different fields, different formats — never put `"16:00"` in the seed.
3. **`sanitizeHares` does NOT strip mid-string phones.** Scrub PII (both `01x-…` and `+82 1X-…`, multi-space)
   at backfill freeze **and** in the live adapter; share one `src/` scrubber; guard the committed JSON with a
   regression test; drop bulky `description` from historical rows.
4. **New regex modules:** literals not `new RegExp`, one-per-type (S5843 < 20), `\d` not `[0-9]`, `.exec` not
   `.match`, backtracking-safe email, string-op cleanup; mark linear-but-flagged S5852 hotspots SAFE via API.
5. **Collision check = EVERY bare initialism** in the alias list (`SH3` *and* `SHHH`), not just the obvious one.
6. **Verify sub-agent "file X contains Y" claims against the code** before dropping/adding a planned edit.
7. **Keep:** the `▶ FOR CLAUDE CODE` directive, single-block fail-loud + `upcomingOnly`, the 5-edit
   new-country `region.ts` pattern, the `manila-h3` reference, coord-sanity (no default-pin), `title`-undefined
   synthesis, and the frozen-dataset archive backfill.
