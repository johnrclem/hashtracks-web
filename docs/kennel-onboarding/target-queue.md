# Kennel Onboarding тАФ Target Queue

Ranked backlog for the daily onboarding task. **Rank 1 = onboard next.** The daily run
([`daily-onboarding-prompt.md`](daily-onboarding-prompt.md)) takes the top `queued` row each
morning. When `queued` rows drop to **5 or fewer**, the run refills the list back to ~20.

**Inclusion rule:** every entry must have a **confirmed/identified dynamic data source**
(Google Calendar, iCal, Meetup, Harrier Central, Hash Rego, Google Sheets, or a scrapeable website)
that is **live and recently active** тАФ upcoming events OR a regular cadence whose latest run is
within ~2├Ч its interval (the *recently-active* rule; 0 upcoming alone is fine). No Facebook-only /
static-only kennels.

**Dedup rule (see prompt Step 2) тАФ against the LIVE sitemap, read via the Chrome MCP.** The seed
files are incomplete (live site has 410+ kennel slugs in the sitemap vs ~349 in seed; kennels are
added straight to prod via the admin UI), the prod DB isn't reachable from the run's sandbox, and
the kennel directory is client-rendered. The authoritative oracle is `hashtracks.xyz/sitemap.xml`
read through the Chrome MCP. **Slugs aren't always the obvious kebab** тАФ scan for fragments
(Singapore Harriets = `sg-harriets`, Miami Valley = `mvh3-day`, HK Monday = `hk-h3`, Brussels
Manneke Piss = `bmph3`).

## How this doc is organized

- **[Active queue](#active-queue-queued--onboard-next) (`queued`)** тАФ ready to onboard now; the daily run takes the top row.
- **[Needs source verification](#needs-source-verification-verify-live-first) (`verify-live-first`)** тАФ promising kennels whose live dynamic source must be confirmed (often via browserRender/Chrome) before they can be queued.
- **[Handed off тАФ awaiting implementation](#handed-off--awaiting-implementation)** тАФ research + handoff doc complete; awaiting Claude Code build/PR.
- **[Shipped / live in prod](#shipped--live-in-prod)** тАФ fully onboarded (`shipped` or `done (already live)`); reference only.
- **[Blocked / dormant](#blocked--dormant)** тАФ verified no live dynamic source; re-check on refill.
- **[Leads](#leads--verify-a-live-dynamic-source-before-queueing) ┬╖ [Removed (already live)](#removed--already-live-authoritative-sitemap-check) ┬╖ [Parking lot](#parking-lot--rejected-no-usable-dynamic-source)** тАФ candidate bench, dedup exclusions, and rejects.
