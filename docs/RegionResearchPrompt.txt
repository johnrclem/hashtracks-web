## Phase 1: Preparation & Deduplication

Before diving into web research, review your provided context:

### kennelCode Collision Check (CRITICAL)

`kennelCode` is **globally unique, lowercase, URL-safe, and permanent**. Collisions are fatal errors in seed.

- **Always search existing kennelCodes in `prisma/seed.ts` before assigning new ones.**
- Common collisions: "ch3" (Chicago vs Charleston), "ph3" (Philly vs Pinelake), "wh3" (multiple)
- Convention: use region suffix for disambiguation — `ch3-sc` (Charleston), `ph3-atl` (Atlanta), `ech3-fl` (Florida)
- `shortName` CAN duplicate across regions (display only) — `kennelCode` CANNOT

### Active Kennel Gate (HARD REQUIREMENT)

**Only include kennels with evidence of activity within the last 6 months.** This is NOT optional — the project already has too many inactive/defunct kennels and cleaning up dead data wastes time.

**Acceptable evidence of activity:**
- Recent Facebook posts about runs (within 6 months)
- Active Meetup group with upcoming or recent events
- Updated website hareline with future events
- Google Calendar with future events
- Recent Instagram posts about trail activity

**Red flags — SKIP the kennel:**
- Dead website (DNS error, parked domain, 404)
- No Facebook posts in 6+ months
- "On hiatus", "inactive", "dissolved"
- No upcoming events anywhere online
- Last Meetup event > 6 months ago
- Only evidence is historical mentions on other sites

**When in doubt, leave it out.** It's easy to add a kennel later, but cleaning up dead data wastes time.

### 2. Source Assessment

Evaluate ALL potential data sources for each kennel. Classify them by adapter type with correct technical details:

**GOOGLE_CALENDAR** (trustLevel 7, scrapeDays 365, scrapeFreq "daily")
- URL is the calendar **email address** (e.g., `bostonhash@gmail.com`, `...@group.calendar.google.com`), NOT the iframe embed URL
- Extract from iframe `src` → find the `src=` param → URL-decode it → that's the calendar ID/email
- Multi-kennel calendars need `kennelPatterns` config (regex→tag tuples on SUMMARY field)
- Single-kennel calendars use `defaultKennelTag`
- Regional aggregator calendars are the highest-value onboarding target (one source = many kennels)

**MEETUP** (trustLevel 7, scrapeDays 90, scrapeFreq "daily")
- Extract `groupUrlname` from URL path: `meetup.com/<groupUrlname>/`
- No API key needed — uses Meetup's public REST API
- Config needs `groupUrlname` and `kennelTag`

**ICAL_FEED** (trustLevel 7-8, scrapeDays 180, scrapeFreq "daily")
- URL of .ics feed
- Supports `kennelPatterns` (regex→tag tuples on SUMMARY), `skipPatterns`, `defaultKennelTag`

**HASHREGO** (trustLevel 8, scrapeDays 90, scrapeFreq "daily")
- `kennelSlugs` are Hash Rego's **internal IDs** (e.g., "BFMH3", "EWH3"), NOT HashTracks kennelCodes
- Multi-kennel source — one source entry covers all linked kennels

**GOOGLE_SHEETS** (trustLevel 7, scrapeDays 9999, scrapeFreq "daily")
- Needs `sheetId`, `columns` mapping, `kennelTagRules`
- Config-driven, no code changes needed

**HTML_SCRAPER** (trustLevel 5-8, scrapeDays 90-365, scrapeFreq "daily")
- Static HTML → Cheerio parsing (most sites)
- JS-rendered (Wix, Google Sites, SPAs) → NAS headless browser rendering service
- Special patterns: Blogger API v3 (Blogspot sites), WordPress REST API, Ghost Content API, PHP REST API (FullCalendar)
- `GenericHtmlAdapter` handles config-driven CSS selector scraping (no code)
- Custom adapter code needed for complex layouts

**STATIC_SCHEDULE** (trustLevel 3, scrapeDays 90, scrapeFreq "weekly")
- For Facebook-only kennels with predictable schedules — generates placeholder events, NO external fetch
- Config uses `kennelTag` and `startTime` (NOT `defaultKennelTag` / `defaultStartTime`)
- `anchorDate` (YYYY-MM-DD) required when INTERVAL > 1 (biweekly) — a known past occurrence date
- Supported RRULE patterns:
  - `FREQ=WEEKLY;BYDAY=SA` — every Saturday
  - `FREQ=WEEKLY;INTERVAL=2;BYDAY=SA` — biweekly Saturday (needs `anchorDate`)
  - `FREQ=MONTHLY;BYDAY=1SU` — 1st Sunday of month
  - `FREQ=MONTHLY;BYDAY=-1FR` — last Friday of month
  - `FREQ=MONTHLY;BYMONTHDAY=15` — 15th of month
- Limitation: cannot express lunar recurrence (full/new moon schedules)

### 3. Source Recommendation

Recommend the **best primary source** and any secondary sources. Preference order (based on actual success rates):

1. `GOOGLE_CALENDAR` — cleanest API, best for aggregators (one source covers 10+ kennels)
2. `MEETUP` / `ICAL_FEED` / `HASHREGO` — config-driven, no code, rich data
3. `GOOGLE_SHEETS` — config-driven, no code
4. `HTML_SCRAPER` — requires adapter code or GenericHtmlAdapter config
5. `STATIC_SCHEDULE` — fallback for Facebook-only (placeholder data, not real events)
6. No source — kennel-only record, no events on hareline

*Crucial:* If multiple kennels share a source (regional aggregator pattern), note this explicitly!

### STATIC_SCHEDULE Conventions

Standard field conventions for Facebook-only / schedule-only kennels:

- `defaultTitle`: `"<Kennel Full Name> <Frequency> Run"` — e.g., "Wildcard H3 Weekly Run", "Lowcountry H3 Biweekly Run", "Full Moon H3 Monthly Run"
- `defaultDescription`: `"Check the Facebook page at <facebookUrl> for updates on locations."` — must include the actual Facebook URL so users can find details
- `defaultLocation`: City or metro area — e.g., "Fort Lauderdale, FL", "Greenville, SC"

These are trustLevel 3 placeholder events — the description directing users to Facebook is critical since we don't have real event details.

### Multi-Source Strategy

- When a kennel has Calendar AND website → add BOTH (Calendar trustLevel 7, website trustLevel 8 for primary site)
- Merge pipeline deduplicates via fingerprint, uses trust levels for field precedence
- Common: `STATIC_SCHEDULE` (trustLevel 3) as fallback + `MEETUP` (trustLevel 7) when available
- Regional aggregator Calendars are highest-value targets — one source = many kennels with zero code
